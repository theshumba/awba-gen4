## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Elevation | node:vm ingest of data files | accept | node:vm is not a security sandbox; validator runs only Josh's own repo files at author-time, never third-party/user-uploaded input. Do not market the vm as a security boundary; if that assumption ever changes, move to a separate process (RESEARCH Security Domain). |
| T-02-06 | Denial of Service (self) | ingest() of a malformed/no-inline-script file | mitigate | wrap ingest in try/catch; a file with no inline script or a throwing cfg becomes a reported error, not a crash — the gate reports every file cleanly and still exits non-zero |
| T-02-07 | Tampering (false-negative) | id-resolution via JSON.stringify | mitigate | collect ids from RAW strings, never JSON.stringify (Pitfall 2) — stringify escapes quotes and silently finds zero ids, passing broken files |
| T-02-SC | Tampering (supply chain) | npm/pip installs | not-applicable | zero external packages by D-25; Node core only; no `package.json`, no install task, no registry surface to slopcheck. Acceptance gate asserts only `node:` requires. |
