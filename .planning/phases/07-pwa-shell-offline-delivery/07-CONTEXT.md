## Decisions

**D-69 — the app icon binds "lantern icon family" to the surviving Athar lantern.** PLT-02/ROADMAP say "lantern icon family," and Athar retired only the lantern-*gold* colour variant (D-A6) and the lantern *companion mascot* — the lantern **scene icon** is still a first-class currentColor member of AW.KIT. So the PWA icon is that lantern, given an Athar home-screen treatment: **gold ink (`--gold`) on the Kiswah dark ground (`#131013`, the shipped dark `theme-color`)**, generated as PNG at **192×192, 512×512, a 512 maskable (icon art ≤ 60% of the frame → ≥40% safe zone), and 180×180 apple-touch** (iOS ignores the manifest for the home-screen icon and reads `apple-touch-icon`; no transparency, per CLAUDE.md). Generated by **rasterising an Athar-treated SVG through headless system-Chrome** (the same zero-dependency tool render-smoke uses) — no new npm/dependency. *(Visual treatment is an owner-ledger walk item; the mechanism is fixed.)*

**D-70 — `index.html` is an instant redirect to `learn.html`; the app is not renamed.** `learn.html` is referenced across the engine, the VT nav probe, and every internal link — renaming it is high-blast-radius for zero benefit. `index.html` (what GitHub Pages serves at the base) is a tiny page that redirects to `learn.html` (meta-refresh + `location.replace` + a plain link fallback), carrying the manifest link + dark `theme-color` + `apple-touch-icon` so it is installable if a user hits the bare URL. Manifest `start_url` = `learn.html`, `scope` = `./` (**relative** — no leading slash — so it works at a GitHub Pages project subpath `…/awba-gen4/` OR a custom-domain root, matching the build's absolute no-leading-slash law).

**D-71 — one hand-written root-scoped service worker (`sw.js` at repo root), < 50 lines.** Registered (relative path) from `learn.html` + `index.html` behind `'serviceWorker' in navigator` + a `location.protocol !== 'file:'` guard (SW never registers over `file://` — CLAUDE.md — so double-click review is unaffected). **Cache-first** for static/hashed assets (CSS/JS/fonts/img/icons/pages in the precache set); **network-first** for HTML navigations (fresh content when online, the cached page when offline); **`activate`-time purge** of any cache key ≠ the current `awba-cache-vN` (manual version bump busts stale assets). Precache list = the 20 app pages + engine CSS/JS + 17 fonts + grain.png + the 4 icons. Same-origin only (`request.method === 'GET'` + same origin); cross-origin/opaque requests pass straight through.

**D-72 — a mercy-toned, dismissible add-to-home nudge on `learn.html`.** Android/Chromium: capture `beforeinstallprompt` (preventDefault, stash it), show a gentle inline banner ("Keep Awba a tap away — add it to your home screen"), fire the stashed prompt on accept. iOS/iPadOS Safari (no `beforeinstallprompt`): a one-line hint to use Share → *Add to Home Screen*. **Never nagging** — shown at most once per install-state, and **dismissal is remembered through `AW.prefs`** (the engine's storage wrapper — so `learn.html`'s own `localStorage` grep stays 0; a new pref key, no schema bump to the state blob). Honors dismissal; re-appears only if still uninstalled after a long gap (or not at all). Copies never say "free"/"upgrade"/"unlock".

**D-73 — `README.md` at repo root (PLT-04).** Covers: the structure (one engine CSS + one engine JS + per-page data files; the `shared/` layout), how Josh reviews (open any `.html` directly via `file://` — no server, no build), how to run the content validator (`node scripts/validate-content.js …`) + the standing gates, how to test the PWA locally (`npx serve` / `python3 -m http.server` — the SW needs `http://localhost`, never `file://`), and how to deploy (GitHub Pages: push, serve static, zero build).

**D-74 — the PLT-05 Gen-3 v1.1–v1.5 regression ship-gate is a documented checklist verified against the shipped code + tests before close.** Items → where verified: popup anchoring (`learn.html` `.npop` `placePop` edge-clamp + the a11y-dialogs probe), footer/hero spacing (lesson `@layer screens` `.foot`/`.hero` + render-smoke), review timer teeth (`AwbaReview` 14s + `.low` + `runner-review` tests), accordion lenses (`.lens.open > .lb` + components tests), chest idempotency (`__awbaClaimChest` write-once + the +25-survives-Escape test), TF selection visibility (the ACC-03 `aria-pressed` + 3px cue + contrast-audit), back-button rules (no back button in reviews; lesson back advances — pinned by `runner-interaction`). New assertions added only where a gap is found.

**D-75 — deploy = GitHub Pages, static, zero build, relative paths** ("keep it simple"). No CI; the repo IS the deploy artifact. `awba.app`/custom domain is an owner DNS follow-up — relative paths mean no code change when the base moves.
