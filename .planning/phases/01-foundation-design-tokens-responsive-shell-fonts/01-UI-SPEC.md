## Spacing Scale

Declared values (multiples of 4). CSS tokens `--sp-*` in the `tokens` layer. Gen-3's legacy odd values (13, 14, 22, 9, 11 px) normalize to the nearest step below.

| Token | Value | Usage |
|-------|-------|-------|
| `--sp-2` | 2px | Hairline only (icon nudge, 1.5pxв†’2px borders) вЂ” exception |
| `--sp-1` (xs) | 4px | Icon gaps, chip inline padding, inline stat gaps |
| `--sp-2s` (sm) | 8px | Compact stacking (option gaps, tile gaps, HUD gaps) |
| `--sp-3` (md-) | 12px | Card inner padding, marker padding, list rhythm |
| `--sp-4` (md) | 16px | Default element spacing, button padding, footer padding |
| `--sp-5` (lg-) | 20px | Between-block spacing, sheet body rhythm |
| `--sp-6` (lg) | 24px | Stage horizontal padding, section padding (Gen-3's 22 в†’ 24) |
| `--sp-8` (xl) | 32px | Layout gaps, hero vertical rhythm |
| `--sp-12` (2xl) | 48px | Major section breaks, hero top/bottom |
| `--sp-16` (3xl) | 64px | Page-level spacing, desktop column gutters |

**Exceptions:**
- `44px` minimum interactive touch target (iOS HIG) вЂ” icon-only controls (HUD close/back, tab items) meet 44Г—44 hit area even when the visual glyph is smaller.
- `env(safe-area-inset-*)` вЂ” dynamic device insets added ON TOP of the scale (HUD top, footer/tab bottom); never hard-coded.
- `2px` hairline (above) for the gummy-button hard shadow offset base unit.

## Typography

**Families** (self-hosted, CSS vars): `--font-disp:'Poppins'` В· `--font-body:'Inter'` В· `--font-ar:'Amiri'` В· `--font-quran:'Amiri Quran'`. Metric-tuned system fallback stacks so the `font-display:swap` first paint is gentle (D-11).

**Weights (declared, disciplined):** two families, two core weights each, plus one reserved display weight.
- Inter (body): **400** (regular) + **600** (semibold, UI emphasis / terms / labels)
- Poppins (display): **600** (headings, kickers, panel names) + **700** (buttons, stat values, foot titles)
- Poppins **800** вЂ” RESERVED, single use: the giant hero streak/returns numeral only.

> Deviation from the generic "2 weights max" heuristic is deliberate and recorded: a Duolingo-grade gamified app needs display weight contrast between reading body (Inter) and the celebratory display layer (Poppins). Five weights total across two families, each family internally disciplined to two, with 800 quarantined to one element. Rationale: matches Gen-3's proven register split without weight sprawl.

**Type scale** вЂ” fluid via `rem` + `clamp()` where it matters (display, hero numerals); body fixed and comfortably readable at 320px with no overflow (D-03). CSS tokens `--fs-*`.

| Role | Token | Size (min в†’ max) | Family / Weight | Line Height | Notes |
|------|-------|------------------|-----------------|-------------|-------|
| Hero numeral | `--fs-hero` | `clamp(3.5rem, 3rem + 2.5vw, 5.125rem)` (56в†’82px) | Poppins 800 | 1.0 | Streak/returns big number only |
| Display | `--fs-display` | `clamp(2rem, 1.7rem + 1.5vw, 2.375rem)` (32в†’38px) | Poppins 700/800 | 1.05 | Noor count-up, verdict |
| H1 / title | `--fs-h1` | `clamp(1.375rem, 1.25rem + 0.6vw, 1.625rem)` (22в†’26px) | Poppins 600 | 1.16 | Screen titles |
| H2 / point | `--fs-h2` | `clamp(1.125rem, 1.05rem + 0.4vw, 1.25rem)` (18в†’20px) | Poppins 600 | 1.28 | Depth point, quiz question |
| Lead | `--fs-lead` | `1.0625rem` (17px) | Inter 400 | 1.58 | Opening paragraphs, frame lead |

**Arabic & RTL typography laws (binding вЂ” ASSETS.md, CNT-04, D-09/D-10):**
- Every Arabic span carries `lang="ar" dir="rtl"`; Arabic containers use `unicode-bidi: isolate` so mixed Arabic/Latin lines never scramble.
- `.ayah` class binds to **Amiri Quran only** (Qur'an-optimized face) вЂ” reserved for verbatim Qur'an; never used for UI chrome.
- General Arabic (hadith, du'a, terms) uses **Amiri**.
- Arabic body size в‰Ґ **1.4Г— Latin body** в†’ min `1.4rem` (22.4px). `--fs-ar-body: 1.4rem`; `--fs-ayah: clamp(1.5rem, 1.4rem + 0.5vw, 1.6rem)` (24в†’25.6px); sheet Arabic display `2rem` (32px).
- Arabic line-height в‰Ґ **1.8** (accommodates diacritics); ayah `1.9`.
- `letter-spacing: 0` on ALL Arabic вЂ” always. Arabic is a connected script; any tracking breaks glyph joining. Never fake-bold, never letter-spaced.
- п·є (U+FDFA) follows the Prophet's name; honorifics written in full or as the ligature glyph вЂ” never "PBUH"/"SWT" acronyms.

### Per-Unit Accent Scales (FND-02 / D-06 вЂ” full recolor, WCAG-checked)

Each unit defines its FULL scale explicitly in the `tokens` layer вЂ” there is no shared `--blue2`/`--blue3` for a screen to accidentally inherit (that is the structural death of Gen-3's half-theming bug). `color-mix()` is allowed for tints; **deep shades and inks are hand-picked for contrast**. Six semantic slots per unit:

- `--accent` вЂ” identity hue + primary button fill (white/dark label per unit)
- `--accent-deep` вЂ” gummy 3D shadow offset + dark accent text on light surfaces
- `--accent-bright` вЂ” decorative glow / gradient highlight only (non-text; contrast-exempt)
- `--accent-soft` вЂ” pale tint surface (chips, markers, model boxes, tinted panels)
- `--accent-line` вЂ” hairline border on soft/paper surfaces
- `--accent-ink` вЂ” readable text/label ON `--accent-soft`
- `--accent-on` вЂ” text/icon color ON a filled `--accent` surface

## Responsive Shell Anatomy (PLT-01 / D-01, D-02)

**Kills Gen-3's fixed `380Г—788` phone card.** The shell is a CSS grid; the app IS the page.

**Grid:** `min-height: 100dvh`, `display: grid`, `grid-template-rows: auto 1fr auto` в†’
- **HUD** (row 1, `auto` в‰€ 56px + `env(safe-area-inset-top)`) вЂ” course/close, progress segments or lamps, streak stat, noor stat.
- **Stage** (row 2, `1fr`, `overflow-y:auto`, `overscroll-behavior: contain`) вЂ” the scrolling content region.
- **Footer / tab bar** (row 3, `auto` + `env(safe-area-inset-bottom)`) вЂ” primary CTA (lesson/review) or tab bar (learn).

**Mobile (< 600px):** full-bleed edge-to-edge, `100dvh` (dynamic viewport unit вЂ” survives iOS URL-bar collapse), safe-area insets honored on HUD top and footer bottom, `overscroll-behavior: contain` prevents rubber-band bleed, `-webkit-tap-highlight-color: transparent` (compensated by `:focus-visible` below). No `.phone` card, no fixed height.

**Desktop (в‰Ґ 600px):** **intentional centered column** вЂ” `max-width: 460вЂ“480px`, full-height, centered on a designed calm backdrop (cream `#EEF2FB` with a very subtle brand radial wash вЂ” a soft off-white-to-cream gradient, no hard edges). The column carries `--r-2xl` (28px) corners, `--sh-4` elevation, `--sh-inset` top highlight. Column height breathes with the viewport (`min(100dvh - 2 Г— gutter, ...)` вЂ” never a fixed 788px, never a fake phone bezel). A reviewer resizing the window sees graceful adaptation at every width between 320px and desktop.

**Breakpoint behavior:**
- 320вЂ“599px: full-bleed shell, fluid type via `clamp()`.
- в‰Ґ 600px: transition to centered-column presentation; backdrop wash appears; type clamps reach their max.
- No `@custom-media` (not Baseline) вЂ” repeated `@media` queries in the `base` layer.

## Focus-Visible Treatment (ACC-01 foundation)

Tap-highlight is globally suppressed (`-webkit-tap-highlight-color: transparent`) for a clean touch feel вЂ” so **every** interactive element MUST carry a visible `:focus-visible` style (keyboard users are otherwise stranded вЂ” a Gen-3 gap). Defined in the `base` layer now; every component in later phases inherits it.

- Applies to: buttons, `.opt`, `.tf`, `.tile`, chips, `.term`, sheet rows, tabs, nodes, close/back.
- Style: `outline: 3px solid var(--accent); outline-offset: 2px;` + a soft companion ring `box-shadow: 0 0 0 6px var(--accent-soft)`; `border-radius` follows the element's own radius.
- Shown on `:focus-visible` only (keyboard), NOT plain `:focus` (mouse/touch) вЂ” no focus ring flash on tap.
- On the gold/night legendary register, focus ring switches to `--gold2` for visibility against the dark gradient.
- Contrast: focus outline meets в‰Ґ3:1 against both cream `--bg` and the night gradient (verified in Phase 6 ACC-03).
