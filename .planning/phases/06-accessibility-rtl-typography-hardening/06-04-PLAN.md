## Phase Goal

**As a** blind or keyboard-only learner, **I want to** hear the app's calm voice announce what just happened and never get trapped or lost inside a dialog, **so that** the same mercy and clarity a sighted learner gets reaches me through one polite live region and one focus grammar.

*(Derived from ROADMAP Phase-6 SC1/SC2 + 06-CONTEXT §specifics. This plan builds the three shared engine primitives — announce, focus-trap, the visually-hidden utility — plus the sheet accessible name, interface-first, so the runner and learn fixes wire them.)*

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| announced text → live region | `AW.announce` writes composed app strings via `textContent` (never innerHTML) — no markup/script injection into the region |
| overlay DOM → focus trap | `AW._trapFocus` reads the overlay's own focusables and wraps focus; it never reaches outside the overlay or mutates content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
